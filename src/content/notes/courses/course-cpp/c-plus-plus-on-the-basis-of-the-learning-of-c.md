## 6.类型转换
### a.静态转换（$Static Cast$）
+ 将一种数据类型的值**强制转换**为另一种数据类型的值。
+ 通常用于比较**类型相似**的对象之间的转换，例如将 int 类型转换为 float 类型。
+ **不进行**任何运行时类型**检查**，因此可能会导致运行时错误。
+ 实例：`float f = static_cast<float>(i); // 静态将int类型转换为float类型`
### b.动态转换（$Dynamic Cast$）
+ 用于在继承层次结构中进行向下转换（$downcasting$）
+ 通常用于将一个基类指针或引用转换为派生类指针或引用
+ 在运行时进行类型检查。如果转换失败，指针类型返回 `nullptr`，引用类型抛出 `std::bad_cast` 异常
+ **语法：**`dynamic_cast<目标类型>(表达式)`
### c.常量转换（$Const Cast$）
+ 用于将 const 类型的对象转换为非 const 类型的对象
+ 只能转换掉 const 属性，**不能**改变对象的类型
+ 实例：`const int i = 10; int& r = const_cast<int&>(i);`
### d.重新解释转换（$ReinterpretCast$）
+ 将一个数据类型的值重新解释为另一个数据类型的值，通常用于不相关的数据类型之间进行转换
+ 不进行任何类型检查，可能会导致未定义的行为
+ 实例：`int i = 10; float& f = reinterpret_cast<float&>(i); // 将int转换为float`

# 三、常量

## 1. 整数常量
1. **前缀**：$0x$ 或 $0X$ 表示十六进制，$0$ 表示八进制，不带前缀默认十进制。
2. **后缀**：U 表示无符号整数（unsigned），L 表示长整数（long），可大写可小写，U 和 L 的**顺序任意**。
3. 常见错误：
    - 7UU 不能重复后缀
    - 078 非八进制

## 2. 浮点常量

1. **形式**：必须有小数点，整数和小数部分取其一
2. 两种方式：
    - **指数**表示：使用$e$或$E$，整数部分可任意位
    - **小数**表示
3. **后缀**：$f/F$表示$float$,$l/L$表示$long\ double$,默认$double$

## 3. 布尔常量

1. 都为标准**C++ 关键字**（不需要stdbool库）
    - **true** 值代表真。
    - **false** 值代表假。
2. 不应把 true 的值看成 1，把 false 的值看成 0。
    - 模糊真值含义
    - 转换困难

## 2. 引用

1. **概念**：引用不是新定义一个变量，而是给已存在变量**取了一个别名**，编译器不会为引用变量开辟内存空间，它和它引用的变量**共用同一块内存空间**
2. **语法**：类型& 引用变量名(对象名) = 引用实体；
```c++
int a = 10;
int& b = a;  // b 是 a 的别名
```
3. **特性**:
	+ 引用在定义时**必须初始化**
	+  一个变量可以有**多个引用**
	+  一个引用可以**继续有引用**，也同属于一块空间，但不能有**引用的引用**：`int& (&a)`
	+  引用一旦引用一个实体，再**不能引用其他实体**，即**不能重新绑定**
	+  可以对**任何类型**做引用（变量、指针…）
	+ **不占用**额外内存
	+ **直接**使用，无需解引用操作符
# 六、C++ 函数

## 1. 调用函数
有三种向函数传递参数的方式：
+ **传值**调用
+ **指针**调用
+ **引用**调用
```c++
	//交换a,b
	void swap(int &x, int &y)
	{
	   int temp;
	   temp = x; /* 保存地址 x 的值 */
	   x = y;    /* 把 y 赋值给 x */
	   y = temp; /* 把 x 赋值给 y  */

	   return;
	}
```
## 2. 参数的默认值
1. 定义一个函数时，可以为参数列表中后边的每一个参数**指定默认值**。当调用函数时，如果实际参数的值留空，则使用这个默认值。
	+ e.g. `int sum(int a, int b=20){...}`
2.  **使用规则** :
	+ **半缺省参数**必须从**右往左依次且连续**来给出，不能间隔
	+ 缺省参数**不能**在函数声明和定义中**同时出现**
	+ 缺省值必须是**常量**或者**全局变量**，一般为**常量**
## 3. Lambda 函数(匿名函数)
+ **具体形式**：`[捕获列表](参数列表) -> 返回类型 { 函数体 }`
	+ **捕获列表**：用于捕获外部变量，决定 Lambda 函数如何访问外部作用域的变量。
	+ **参数列表**：与普通函数的参数列表类似。
	+ **返回类型**：可以省略，编译器会自动推导。
	+ **函数体**：Lambda 函数的实现代码。
+ **捕获列表**
	- `[]`：不捕获任何外部变量。
	- `[=]`：以值的方式捕获所有外部变量。
	- `[&]`：以引用的方式捕获所有外部变量。
	- `[var]`：以值的方式捕获特定变量 `var`。
	- `[&var]`：以引用的方式捕获特定变量 `var`。
	- `[=, &var]`：以值的方式捕获所有外部变量，但以引用的方式捕获 `var`。
	- `[&, var]`：以引用的方式捕获所有外部变量，但以值的方式捕获 `var`。
+ `this`指针
	+ 对于`[=]`或`[&]`的形式，`lambda` 表达式可以直接使用` this` 指针
	+ 但是，对于`[]`的形式，如果要使用` this `指针，必须显式传入:
		`[this]() { this->someFunc(); }();`

# 九、C++ 结构体(struct)
## 1. 结构体与类的区别

在 C++ 中，struct 和 class 本质上非常相似，唯一的区别在于默认的访问权限：

- `struct` 默认的成员和继承是 `public`。
- `class` 默认的成员和继承是 `private`。

可以将 `struct` 当作简化形式的 `class`，适合用于没有太多复杂功能的简单数据封装。
## 2. 结构体与函数的结合

+ 可以通过构造函数初始化结构体
+ 可以通过**引用传递结构体**来避免不必要的拷贝。
```c++
struct Books {
    string title;
    string author;
    string subject;
    int book_id;

    // 构造函数
    Books(string t, string a, string s, int id)
        : title(t), author(a), subject(s), book_id(id) {}

    void printInfo() const {
        cout << "书籍标题: " << title << endl;
        cout << "书籍作者: " << author << endl;
        cout << "书籍类目: " << subject << endl;
        cout << "书籍 ID: " << book_id << endl;
    }
};

void printBookByRef(const Books& book) {
    book.printInfo();
}
```
# 十、C++ vector 容器
## 1. 基本介绍

1. 是一种序列容器，允许在运行时**动态地插入和删除**元素
2. 基于数组的数据结构，但可以**自动管理内存**
3. 基本特性:
	- **动态大小**：`vector` 的大小可以根据需要自动增长和缩小。
	- **连续存储**：`vector` 中的元素在内存中是连续存储的，这使得访问元素非常快速。
	- **可迭代**：`vector` 可以被迭代，你可以使用循环（如 `for` 循环）来访问它的元素。
	- **元素类型**：`vector` 可以存储任何类型的元素，包括内置类型、对象、指针等。
## 2. 使用

1. 需要包含`<vector>` **头文件** `#include <vector>`
2. **创建** vector :
	+ 方法一：`vector<int> list1;` **默认初始化，vector 为空， size 为0**
	+ 方法二：`vector<int> list2(list);`或`vector<int> list2 = list;`,`list2` 初始化为 `list` 的**拷贝**
	+ 方法三：`vector<int> list = {1,2,3.0,4,5,6,7};`或`vector<int> list {1,2,3.0,4,5,6,7};`,`list` 初始化为**列表中元素的拷贝**，列表中元素必须与 `list` 的元素**类型相容**
	+ 方法四：`vector<int> list(list.begin() + 2, list.end() - 1);` `list` 初始化为**两个迭代器指定范围中元素的拷贝**，范围中的元素类型必须与 `list` 的元素**类型相容**
	+ 方法五：`vector<int> list(n);`**默认值初始化**，`list` 中将包含$n$个元素，每个元素进行**缺省的值初始化** (int 为 0)
	+ 方法六：`vector<int> list(n, m)` **指定值初始化**，`list` 被初始化为包含n个值为m的`int`
3. **添加元素** : `push_back` 方法 `myVector.push_back(n); // 将 n 添加到 vector 的末尾`
4. **访问元素** : 使用下标操作符` [] `或 `at() `方法`int x = myVector[0];int y = myVector.at(1);`
5. **获取大小** :  `size()` 方法 `int size = myVector.size();`
6. **迭代访问** ：
	+ 使用迭代器遍历 ：`for (auto it = myVector.begin(); it != myVector.end(); ++it) {std::cout << *it << " ";}`
	+ 使用范围循环 : `for (int element : myVector) {std::cout << element << " ";}`
7. **删除元素** : `erase()` 方法 : `myVector.erase(myVector.begin() + 2); // 删除第三个元素`
8. **清空 Vector** :  `clear()` 方法 : `myVector.clear();`
9. **嵌套** ： `vector<vector<int>> outer; // 二维`
# 十一、C++ 数据结构
## 1. **[数组(Array)](/blog/notes/courses/course-data-structures-algorithms/%E6%95%B0%E6%8D%AE%E7%BB%93%E6%9E%84-hello-%E7%AE%97%E6%B3%95)**

1. 数组是最基础的数据结构，用于存储一组相同类型的数据。
2. **特点**：
	- 固定大小，一旦声明，大小不能改变。
	- 直接访问元素，时间复杂度为 O(1)。
	- 适合处理大小已知、元素类型相同的集合。
3. 实例
	```c++
	int arr[5] = {1, 2, 3, 4, 5};
	cout << arr[0]; // 输出第一个元素
	```
4. **优缺点**：
	- 优点：访问速度快，内存紧凑。
	- 缺点：大小固定，无法动态扩展，不适合处理大小不确定的数据集。

## 2. **结构体（Struct）**

1. 结构体允许将不同类型的数据组合在一起，形成一种自定义的数据类型。
2. **特点**：
	- 可以包含不同类型的成员变量。
	- 提供了对数据的基本封装，但功能有限。
3. 实例
	```c++
	struct Person {
	    string name;
	    int age;
	};
	Person p = {"Alice", 25};
	cout << p.name << endl; // 输出 Alice
	```

## 3. **类（Class）**

1. 类是 C++ 中用于面向对象编程的核心结构，允许定义成员变量和成员函数。与 `struct` 类似，但功能更强大，支持**继承、封装、多态**等特性。
2. **特点**：
	- 可以包含**成员变量、成员函数、构造函数、析构函数**。
	- 支持**面向对象特性**，如封装、继承、多态。
3. 实例
	```c++
	class Person {
	private:
	    string name;
	    int age;
	public:
	    Person(string n, int a) : name(n), age(a) {}
	    void printInfo() {
	        cout << "Name: " << name << ", Age: " << age << endl;
	    }
	};
	Person p("Bob", 30);
	p.printInfo(); // 输出: Name: Bob, Age: 30
	```

## 4. **[链表(Linked List)](/blog/notes/courses/course-data-structures-algorithms/%E6%95%B0%E6%8D%AE%E7%BB%93%E6%9E%84-hello-%E7%AE%97%E6%B3%95)**

1. 链表是一种动态数据结构，由一系列节点组成，每个节点包含数据和指向下一个节点的指针。
2. **特点**：
	- 动态调整大小，不需要提前定义容量。
	- 插入和删除操作效率高，时间复杂度为 O(1)（在链表头部或尾部操作）。
	- 线性查找，时间复杂度为 O(n)。
3. 实例（单向链表）
	```c++
	struct Node {
	    int data;
	    Node* next;
	};
	Node* head = nullptr;
	Node* newNode = new Node{10, nullptr};
	head = newNode; // 插入新节点
	```
4. **优缺点**：
	- 优点：动态大小，适合频繁插入和删除的场景。
	- 缺点：随机访问效率低，不如数组直接访问快。

## 5. **栈（Stack）**

1. 栈是一种后进先出（LIFO, Last In First Out）的数据结构，常用于递归、深度优先搜索等场景。
2. **特点**：
	- 只允许在栈顶进行插入和删除操作。
	- 时间复杂度为 O(1)。
3. 实例
	```c++
	stack<int> s;
	s.push(1);
	s.push(2);
	cout << s.top(); // 输出 2
	s.pop();
	```
4. **优缺点**：
	- 优点：操作简单，效率高。
	- 缺点：只能在栈顶操作，访问其他元素需要弹出栈顶元素。

## 6. **队列（Queue）**

1. 队列是一种先进先出（FIFO, First In First Out）的数据结构，常用于广度优先搜索、任务调度等场景。
2. **特点**：
	- 插入操作在队尾进行，删除操作在队头进行。
	- 时间复杂度为 O(1)。
3. 实例
	```c++
	queue<int> q;
	q.push(1);
	q.push(2);
	cout << q.front(); // 输出 1
	q.pop();
	```
4. **优缺点**：
	- 优点：适合按顺序处理数据的场景，如任务调度。
	- 缺点：无法随机访问元素。

## 7. **双端队列（Deque）**

1. 双端队列允许在两端进行插入和删除操作，是栈和队列的结合体。
2. **特点**：
	- 允许在两端进行插入和删除。
	- 时间复杂度为 O(1)。
3. 实例
	```c++
	deque<int> dq;
	dq.push_back(1);
	dq.push_front(2);
	cout << dq.front(); // 输出 2
	dq.pop_front();
	```
4. **优缺点**：
	- 优点：灵活的双向操作。
	- 缺点：空间占用较大，适合需要在两端频繁操作的场景。

## 8. **哈希表（Hash Table）**

1. 哈希表是一种通过键值对存储数据的数据结构，支持快速查找、插入和删除操作。C++ 中的 `unordered_map` 是哈希表的实现。
2. **特点**：
	- 使用哈希函数快速定位元素，时间复杂度为 O(1)。
	- 不保证元素的顺序。
3. 实例
	```c++
	unordered_map<string, int> hashTable;
	hashTable["apple"] = 10;
	cout << hashTable["apple"]; // 输出 10
	```
4. **优缺点**：
	- 优点：查找、插入、删除操作效率高。
	- 缺点：无法保证元素顺序，哈希冲突时性能会下降。

## 9. **映射（Map）**

1. `map` 是一种有序的键值对容器，底层实现是红黑树。与 `unordered_map` 不同，它保证键的顺序，查找、插入和删除的时间复杂度为 O(log n)。
2. **特点**：
	- 保证元素按键的顺序排列。
	- 使用二叉搜索树实现。
3. 实例
	```c++
	map<string, int> myMap;
	myMap["apple"] = 10;
	cout << myMap["apple"]; // 输出 10
	```
4. **优缺点**：
	- 优点：元素有序，适合需要按顺序处理数据的场景。
	- 缺点：操作效率比 `unordered_map` 略低。

## 10. **集合（Set）**

1. `set` 是一种用于存储唯一元素的有序集合，底层同样使用红黑树实现。它保证元素不重复且有序。
2. **特点**：
	- 保证元素的唯一性。
	- 元素自动按**升序排列**。
	- 时间复杂度为 $O(\log n)$。
3. 实例
	```c++
	set<int> s;
	s.insert(1);
	s.insert(2);
	cout << *s.begin(); // 输出 1
	```
4. **优缺点**：
	- 优点：自动排序和唯一性保证。
	- 缺点：插入和删除的效率不如无序集合。

## 11. **动态数组（Vector）**

1. `vector` 是 C++ 标准库提供的动态数组实现，可以动态扩展容量，支持随机访问。
2. **特点**：
	- 动态调整大小。
	- 支持随机访问，时间复杂度为 O(1)。
	- 当容量不足时，动态扩展，时间复杂度为摊销 O(1)。
3. 实例
	```c++
	vector<int> v;
	v.push_back(1);
	v.push_back(2);
	cout << v[0]; // 输出 1
	```
4. **优缺点**：
	- 优点：支持随机访问，动态扩展。
	- 缺点：插入和删除中间元素的效率较低。

3. 访问数据成员
	+ 类的对象的公共数据成员可以使用直接成员访问运算符 `.` 来访问。
## 2. 类访问修饰符
关键字 `public`、`private`、`protected` 称为访问修饰符,一个类可以有多个 `public`、`private`、`protected`  标记区域,每个标记区域在下一个标记区域开始之前或者在遇到类主体结束右括号之前都是有效的

1. **默认**访问修饰符是 `private`
2. **公有（`public`）成员** :
	+ 类的外部可访问
	+ 可以**不使用任何成员函数**来**设置和获取**公有变量的值
3. **私有（`private`）成员** :
	+ 在类的外部是**不可访问，不可查看**
	+ 只有**类和友元函数**可以访问
	+ **默认**情况下，类的所有成员都是私有的
	+ 一般会在**私有区域定义数据**，在**公有区域定义相关的函数**
4. **受保护（`protected`）成员** :
	+ 与私有成员相似，但在**派生类（子类**）中是**可访问**的
5. **继承中的特点**：
	+ **`private` 继承**：基类 `public` 成员，`protected` 成员，`private` 成员的访问属性在派生类中分别变成：`public`, `protected`, `private`
	- **`protected` 继承**：基类 `public` 成员，`protected` 成员，`private` 成员的访问属性在派生类中分别变成：`protected`, `protected`, `private`
	- **`private` 继承**：基类 `public` 成员，`protected` 成员，`private` 成员的访问属性在派生类中分别变成：`private`, `private`, `private`
	- 无论哪种继承方式，下面两点都没有改变：
		- 1.`private` 成员只能被本类成员（类内）和友元访问，不能被派生类访问；
		- 2.`protected` 成员可以被派生类访问。

| 继承方式        | 基类的public成员   | 基类的protected成员 | 基类的private成员 | 继承引起的访问控制关系变化概括     |
| ----------- | ------------- | -------------- | ------------ | ------------------- |
| public继承    | 仍为public成员    | 仍为protected成员  | 不可见          | 基类的非私有成员在子类的访问属性不变  |
| protected继承 | 变为protected成员 | 仍为protected成员  | 不可见          | 基类的非私有成员都为子类的保护成员   |
| private继承   | 变为private成员   | 变为private成员    | 不可见          | 基类中的非私有成员都称为子类的私有成员 |
## 3. 类成员函数
1. 可以操作**类的任意对象**，可以访问**对象中的所有成员**
2. 可以定义在类定义内部，或者单独使用**范围解析运算符 `::`** 来定义
3. 自动将函数声明为**内联** `(inline)`
4.  **`const` 成员函数** :不能修改对象成员
## 4. 类的构造函数
1. 构造函数的名称与类的**名称是完全相同**的，并且**不会返回任何类型**，也**不会返回 `void`**
2. **带参数**的构造函数:在创建对象时就会**给对象赋初始值**
	```c++
	#include <iostream>

	using namespace std;

	class Line
	{
	   public:
	      Line(double len);  // 这是构造函数

	   private:
	      double length;
	};

	// 构造函数
	Line::Line( double len)
	{
	    cout << "Object is being created, length = " << len << endl;
	    length = len;
	}

	int main( )
	{
	   Line line(10.0);

	   return 0;
	}
	```
3. **初始化列表** :
	+ 以**冒号**开头，后跟一系列以**逗号分隔**的初始化字段
		```c++
		ClassName(参数列表) : 成员1(初始值), 成员2(初始值), ... {// 构造函数体}
		```
	+ 成员的初始化顺序与它们在**类中声明的顺序一致**，而**不是初始化列表中的顺序**
	+ **必须使用初始化列表的情况**：
		- 初始化 `const` 成员变量。
		- 初始化引用类型成员变量。
		- 初始化没有默认构造函数的类类型成员变量。
## 5. 类的析构函数
1. 在每次**删除**所创建的对象时执行
2. 名称是在类的名称前面加了个**波浪号~** 作为前缀
3. **不会返回任何值，也不能带有任何参数**
## 6. 拷贝构造函数
1. 用同一类中之前创建的对象来初始化新创建的对象
2. 通常用途：
	- 通过使用另一个同类型的对象来初始化新创建的对象。
	- 复制对象把它作为参数传递给函数。
	- 复制对象，并从函数返回这个对象。
3. 如果类**带有指针变量，并有动态内存分配**，则它必须有一个拷贝构造函数（否则**浅拷贝双重释放**使得系统崩溃）
4. 如果在类中没有定义拷贝构造函数，编译器会自行定义一个（**浅拷贝**）
## 7. 友元函数
1. 定义在类外部,原型在类的定义中出现
2. 有权访问类的所有私有（`private`）成员和保护（`protected`）成员
3. 可以是一个函数（友元函数）；友元也可以是一个类（友元类）
4. 使用关键字 **`friend`**
## 8. 内联函数
1. 在函数名前面放置关键字 **`inline`**
2. 对内联函数进行任何修改，都需要**重新编译**函数的所有客户端，因为编译器需要重新更换一次所有的代码，否则将会继续**使用旧的函数**。
3. 已定义的函数**多于一行**，编译器会**忽略** `inline` 限定符
4. 在类定义中的定义的函数都是内联函数
## 9. this 指针
1. **作用**:
	- **访问当前对象**：在类的非静态成员函数中，`this` 指针指向调用该函数的对象实例
	- **区分成员变量和局部变量**：当成员变量和函数参数或局部变量同名时，可以用 `this->` 明确访问成员变量
	- **链式调用**：通过返回 `*this`，可以实现函数调用的链式操作,如 `obj.func1().func2()`
2.  **特性**:
	+ **隐式存在**：
	    - 在类的非静态成员函数中，编译器自动生成 `this` 指针。
	    - 不需要手动声明，但可以显式使用。
	    - 例如：`void func() { this->x = 10; }`。
	+ **类型**：
	    - `this` 的类型是 `ClassName* const`（指向当前类类型的常量指针）。
	    - 在 `const` 成员函数中，`this` 的类型是 `const ClassName* const`（指向常量的常量指针）。
	+ **不可修改**：
	    - `this` 是一个常量指针，不能修改其指向的地址（如 `this = nullptr;` 是非法的）。
	+ **静态函数无 `this`**：
	    - 静态成员函数（`static`）没有 `this` 指针，因为它们属于类而非对象实例。
	+ **友元函数无 `this`**：
		+ 友元函数没有 **this** 指针，因为友元不是类的成员，只有成员函数才有 **this** 指针

## 11. 类的静态成员
1. **定义**
	- **静态成员变量**：属于类本身，所有对象共享同一份数据。
	- **静态成员函数**：属于类本身，不依赖于对象实例，可以直接通过类名调用。
2. **特点**
	 + **静态成员变量**
		- **共享性**：所有类的实例共享同一个静态成员变量。
		- **内存分配**：静态成员变量在程序启动时分配内存，生命周期直到程序结束。
		- **初始化**：必须在**类外单独初始化**（C++中）。
		- **访问权限**：受类的访问修饰符（`public`/`private`/`protected`）控制。
	+ **静态成员函数**
		- **无 `this` 指针**：不能直接访问非静态成员（变量或函数）
		- **独立于对象**：可以**直接通过类名调用**，无需创建对象
		- **作用**：通常用于操作静态成员变量或实现与类相关的工具函数
3.  **应用场景**
	+ **全局计数器**：统计类的实例数量
	+ **共享资源管理**：例如数据库连接池、日志管理器
	+ **工具函数**：与类相关但不需要对象参与的操作（如数学计算）
	+ **常量定义**：通过 `const static` 定义类内常量（如配置参数）
# 十三、C++继承
## 1. 基类 & 派生类
- 已有的类称为**基类**，新建的类称为**派生类**
- 类派生列表以一个或多个基类命名，形式如下：
	`class derived-class: access-specifier base-class`
- 访问修饰符 access-specifier 是 **`public`、`protected`** 或 **`private`** 其中的一个，`base-class` 是之前定义过的某个类的名称。如果未使用访问修饰符 `access-specifier`，则默认为 `private`
## 2. 访问控制和继承

- 派生类可以访问基类中所有的非私有成员。因此基类成员如果不想被派生类的成员函数访问，则应在基类中声明为 private

| 访问   | public | protected | private |
| ---- | ------ | --------- | ------- |
| 同一个类 | yes    | yes       | yes     |
| 派生类  | yes    | yes       | no      |
| 外部的类 | yes    | no        | no      |
- 一个派生类继承了所有的基类方法，但下列情况**除外**：
	- 基类的构造函数、析构函数和拷贝构造函数
	- 基类的重载运算符
	- 基类的友元函数
- 细节见[继承变化表](/blog/notes/courses/course-cpp/c-plus-plus-on-the-basis-of-the-learning-of-c)
## 3. 多继承
- 一个子类可以有多个父类，它继承了多个父类的特性
- 语法如下
	```c++
	class <派生类名>:<继承方式1><基类名1>,<继承方式2><基类名2>,…
	{
	<派生类类体>
	};
	```
# 十四、C++ 重载运算符和重载函数
- C++ 允许在同一作用域中的某个**函数**和**运算符**指定多个定义，分别称为**函数重载**和**运算符重载**。
- **重载声明**是指一个与之前已经在该作用域内声明过的函数或方法具有相同名称的声明，但是它们的**参数列表和定义（实现）不相同**。
- 当您调用一个**重载函数**或**重载运算符**时，编译器通过把您所使用的参数类型与定义中的参数类型进行比较，决定选用最合适的定义。选择最合适的重载函数或重载运算符的过程，称为**重载决策**
### 1. 函数重载
- 同一个作用域内，可以**声明几个功能类似的同名函数**，但是这些同名函数的**形式参数**（指参数的**个数、类型或者顺序**）**必须不同**
### 2. 运算符重载
- 重载的运算符是带有特殊名称的函数，函数名是由关键字 **operator** 和其后要重载的**运算符符号**构成的。与其他函数一样，重载运算符有一个返回类型和一个参数列表
- 对象作为参数进行传递，对象的属性使用 **`this`** 运算符进行访问
### 3. 可重载运算符/不可重载运算符
| 可重载的运算符 |                                                    |
| ------- | -------------------------------------------------- |
| 双目算术运算符 | `+ ，-，*，/，%`                                       |
| 关系运算符   | `==，!= ，< ，> ，<=，>=`                               |
| 逻辑运算符   | ` \| ，&& ，!`                                       |
| 单目运算符   | `+  ，*(指针)，&(取址)`                                  |
| 自增自减运算符 | `++ ，--`                                           |
| 位运算符    | `\| ，&，~，^,，<<，>>`                                 |
| 赋值运算符   | `=, +=, -=, *=, /= , % = , &=, \| =, ^=, <<=, >>=` |
| 空间申请与释放 | `new, delete, new[ ] , delete[]`                   |
| 其他运算符   | `()(函数调用)，->(成员访问)，,(逗号)，[](下标)`                   |

**不可重载的运算符**列表：

- `.`：成员访问运算符
- `.*, ->*`：成员指针访问运算符
- `::`：域运算符
- `sizeof`：长度运算符
- `?`:：条件运算符
- `#`： 预处理符号
# 十五、C++ 多态
## 1. **虚函数（Virtual Functions）**
- 在基类中声明一个函数为虚函数，使用关键字`virtual`。
- 派生类可以重写$（override）$这个虚函数。
- 调用虚函数时，会根据对象的实际类型来决定调用哪个版本的函数。
- 派生类可以**选择性地重写虚函数**，但**不是必须**
## 2.**动态绑定（Dynamic Binding）**
- 也称为晚期绑定$（Late\ Binding）$，在运行时确定函数调用的具体实现。
- 需要使用指向基类的指针或引用来调用虚函数，编译器在运行时根据对象的实际类型来决定调用哪个函数。
## 3. **纯虚函数（Pure Virtual Functions）**
- 一个包含纯虚函数的类被称为抽象类（$Abstract\ Class$），它不能被直接实例化。
- 纯虚函数没有函数体，声明时使用`= 0`。
- 它**强制**派生类**提供具体的实现**。
- 通常用于定义接口，让派生类实现具体行为
- 至少有一个函数被声明为纯虚函数，则这个类就是**抽象类**
## 4. **多态的实现机制**
- 虚函数表（V-Table）：C++运行时使用虚函数表来实现多态。每个包含虚函数的类都有一个虚函数表，表中存储了指向类中所有虚函数的指针。
- 虚函数指针（V-Ptr）：对象中包含一个指向该类虚函数表的指针。
## 5. **使用多态的优势**
- **代码复用**：通过基类指针或引用，可以操作不同类型的派生类对象，实现代码的复用。
- **扩展性**：新增派生类时，不需要修改依赖于基类的代码，只需要确保新类正确重写了虚函数。
- **解耦**：多态允许程序设计更加模块化，降低类之间的耦合度。
## 6. 虚析构函数
- 如果基类的**析构函数不是虚函数**，那么在删除这个指针时，**只会调用基类的析构函数，而不会调用派生类的析构函数**。这在派生类中有动态分配的内存时会导致内存泄漏
- 如果基类的**析构函数是虚函数**，在删除这个指针时，**会先调用派生类的析构函数，再调用基类的析构函数**
- **构造函数不能是虚函数**，对象还没有实例化，不存在vtable
# 十六、C++ 动态内存
## 1. new 和 delete 运算符
- 与 malloc() 函数相比,`new` 不只是分配了内存，它还创建了对象
- `new`语法：`new data-type;`**data-type** 可以是包括数组在内的**任意内置**的数据类型，也可以是包括类或结构在内的**用户自定义的**任何数据类型
## 2. 数组的动态内存分配
### a. 一维数组
```c++
// 动态分配,数组长度为 m
int *array=new int [m];
//释放内存
delete [] array;
```
### b. 二维数组
```c++
int **array; // 假定数组第一维长度为 m， 第二维长度为 n
// 动态分配空间
array = new int *[m];
for( int i=0; i<m; i++ )
	array[i] = new int [n];
//释放
for( int i=0; i<m; i++ )
	delete [] array[i];
delete [] array;
```
### c. 三维数组
```c++
int ***array;
// 假定数组第一维为 m， 第二维为 n， 第三维为h
// 动态分配空间
array = new int **[m];
for( int i=0; i<m; i++ )
{
    array[i] = new int *[n];
    for( int j=0; j<n; j++ )
    {
        array[i][j] = new int [h];
    }
}
//释放
for( int i=0; i<m; i++ )
{
    for( int j=0; j<n; j++ )
    {
        delete[] array[i][j];
    }
    delete[] array[i];
}
delete[] array;
```
## 3. 对象的动态内存分配
- 如果要为一个包含 $n$ 个 Box 对象的数组分配内存，构造函数将被调用 $n$ 次，同样地，当删除这些对象时，析构函数也将被调用相同的次数（ $n$ 次）
# 十七、C++ 命名空间
## 1. 定义命名空间
1. 本质上，命名空间就是定义了一个范围
2. 定义使用关键字 **namespace**，后跟命名空间的名称
	```c++
	namespace namespace_name {
	   // 代码声明
	}
	```
3. 调用方式：在前面加上命名空间的名称
	`name::code;  // code 可以是变量或函数`
## 2. using 指令
- 使用 **using namespace** 指令，这样在使用命名空间时就可以不用在前面加上命名空间的名称
- 指令会告诉编译器，后续的代码将使用指定的命名空间中的名称,此时，在范围以外定义的**同名实体是隐藏的**
- 可以用来指定**命名空间中的特定项目**，如只打算使用 `std` 命名空间中的 `cout` 部分，可以使用`using std::cout;`
## 3. 生存域
1. **不连续的命名空间**
	- 命名空间**可以定义在几个不同的部分**中，因此命名空间是由几个单独定义的部分组成的。一个命名空间的各个组成部分可以分散在多个文件中
2. **嵌套的命名空间**
	- 命名空间**可以嵌套**，可以在一个命名空间中定义另一个命名空间
	- 可以通过使用 :: 运算符来访问嵌套的命名空间中的成员
	```c++
	namespace namespace_name1 {
	   // 代码声明
	   namespace namespace_name2 {
	      // 代码声明
	   }
	}

	// 访问 namespace_name2 中的成员
	using namespace namespace_name1::namespace_name2;
	// 访问 namespace_name1 中的成员
	using namespace namespace_name1;
	```
3. **全局变量** a 表达为 `::a`，用于当有同名的局部变量时来区别两者
## 4. 命名空间使用
1. **命名空间别名**：简化长命名空间
	当命名空间名过长时，用 `namespace 别名 = 原命名空间名` 简化
2. **命名冲突**（多命名空间同名标识符）
	当多个命名空间有同名标识符时，用**显式作用域解析**或**using声明指定来源**
3. **命名空间组合**：整合多个命名空间
	创建新命名空间，通过 using 声明/指令整合其他命名空间的内容，同时解决冲突
4. **匿名命名空间**：文件内私有
	无名称的命名空间（ `namespace { ... }` ），其内容仅当前文件可见，替代静态全局变量/函数
5. **注意事项**
	1. **避免在头文件中使用 using namespace** ：会导致引入的命名空间污染所有包含该头文件的源文件，引发潜在冲突
	2.  using 声明的**作用域**限制：在函数内的 using 声明仅在函数内有效，在全局作用域的 using 声明会影响整个文件
	3. 命名空间**不支持继承**：命名空间是作用域管理工具，不能像类一样继承
	4. **与类作用域的区别**：类作用域用于封装数据和方法，需实例化（除静态成员）；命名空间仅用于命名管理，无需实例化，可跨文件扩展

# 十九、C++ 异常处理
- **`throw`:** 当问题出现时，程序会抛出一个异常
- **`catch`:** 在您想要处理问题的地方，通过异常处理程序捕获异常
- **`try`** 块中的代码标识将被激活的特定异常。它后面通常跟着一个或多个 `catch` 块
- 示例
	```c++
	try
	{
	   // 保护代码
	}catch( ExceptionName e1 )
	{
	   // catch 块
	}catch( ExceptionName e2 )
	{
	   // catch 块
	}catch( ExceptionName eN )
	{
	   // catch 块
	}
	```
## 1. C++ 标准的异常
- 定义在 `**<exception>`

|异常|描述|
|---|---|
|**std::exception**|该异常是所有标准 C++ 异常的父类。|
|std::bad_alloc|该异常可以通过 **new** 抛出。|
|std::bad_cast|该异常可以通过 **dynamic_cast** 抛出。|
|std::bad_typeid|该异常可以通过 **typeid** 抛出。|
|std::bad_exception|这在处理 C++ 程序中无法预期的异常时非常有用。|
|**std::logic_error**|理论上可以通过读取代码来检测到的异常。|
|std::domain_error|当使用了一个无效的数学域时，会抛出该异常。|
|std::invalid_argument|当使用了无效的参数时，会抛出该异常。|
|std::length_error|当创建了太长的 std::string 时，会抛出该异常。|
|std::out_of_range|该异常可以通过方法抛出，例如 std::vector 和 std::bitset<>::operator[]()。|
|**std::runtime_error**|理论上不可以通过读取代码来检测到的异常。|
|std::overflow_error|当发生数学上溢时，会抛出该异常。|
|std::range_error|当尝试存储超出范围的值时，会抛出该异常。|
|std::underflow_error|当发生数学下溢时，会抛出该异常。|
## 3. 定义新的异常
- 实例
	```c++
	#include <iostream>
	#include <exception>
	using namespace std;

	struct MyException : public exception
	{
	  const char * what () const throw ()
	  {
	    return "C++ Exception";
	  }
	};

	int main()
	{
	  try
	  {
	    throw MyException();
	  }
	  catch(MyException& e)
	  {
	    std::cout << "MyException caught" << std::endl;
	    std::cout << e.what() << std::endl;
	  }
	  catch(std::exception& e)
	  {
	    //其他的错误
	  }
	}
	```

## 一、显卡 GPU
```bash
# 查看显卡状态、显存、占用
nvidia-smi

# 每秒自动刷新看显卡
watch -n 1 nvidia-smi

# 只看GPU 0 占用进程
nvidia-smi --query-compute-apps=pid,used_memory --format=csv
```

## 二、进程管理（查训练、杀任务）
```bash
# 查python训练进程
ps aux | grep python

# 查指定脚本进程
ps aux | grep train_obb.py

# 强制杀进程 PID换成你的进程号
kill -9 PID

# 查看当前终端后台任务
jobs

# 后台任务切前台
fg %序号
```

## 三、磁盘空间 查看/清理
```bash
# 看全盘总容量、已用、可用
df -h

# 看当前目录占用大小
du -sh

# 看当前目录下各子文件夹大小
du -h --max-depth=1

# 查找当前目录大于100M的大文件
find . -type f -size +100M
```

## 四、文件 & 文件夹基础操作
```bash
# 进入目录
cd 文件夹名
cd ..          # 返回上一级
cd ~           # 回到家目录

# 列出文件
ls
ls -lh         # 详细大小、时间

# 新建文件夹
mkdir 文件夹名

# 删除文件
rm 文件名

# 删除文件夹（慎用）
rm -rf 文件夹名

# 复制文件
cp 源文件 目标路径

# 移动/重命名
mv 原名字 新名字
```

## 五、压缩 / 解压
```bash
# 压缩成 tar.gz
tar -zcvf 压缩包名.tar.gz 文件夹名

# 解压 tar.gz
tar -zxvf 压缩包名.tar.gz
```

## 六、日志查看
```bash
# 实时看日志（最常用）
tail -f train_bottle.log

# 看日志最后100行
tail -n 100 train_bottle.log

# 分页看完整日志
less train_bottle.log
# 按 q 退出
```

## 七、nohup 后台训练
```bash
# 标准后台训练模板
nohup python xxx.py > train.log 2>&1 &

# 查看nohup日志
tail -f nohup.out
```

## 八、tmux 会话
```bash
# 新建会话
tmux new -s train1

# 查看所有会话
tmux ls

# 重新连接会话
tmux a -t train1

# 关闭会话
tmux kill-session -t train1
```
**tmux 快捷键**
`Ctrl+b` 松手再按：
- `d` 后台挂起会话
- `c` 新建窗口
- `%` 垂直分屏
- `"` 水平分屏

## 十、SSH 端口隧道（本地连服务器网页）
```bash
# 映射6006 tensorboard
ssh -fN -L 6006:127.0.0.1:6006 用户名@服务器IP

# 映射8888 jupyter
ssh -fN -L 8888:127.0.0.1:8888 用户名@服务器IP

# 关掉后台ssh隧道
ps aux | grep ssh
kill -9 PID
```
1. 假设本机代理端口是 7897，从本机执行：`ssh -N -T -R 127.0.0.1:7897:127.0.0.1:7897 <用户名>@ziplab-5090`
2. 然后在服务器上另开一个终端，设置代理：
	`export http_proxy=http://127.0.0.1:7897`
	`export https_proxy=http://127.0.0.1:7897`
	`export all_proxy=socks5h://127.0.0.1:7897`
3. 测试：`curl -I https://huggingface.co`
4. 如果远端 7897 端口被占用，换成远端 17897：`ssh -N -T -R 127.0.0.1:17897:127.0.0.1:7897 <用户名>@ziplab-5090`
5. 远端对应设置：
	`export http_proxy=http://127.0.0.1:17897`
	`export https_proxy=http://127.0.0.1:17897`
	`export all_proxy=socks5h://127.0.0.1:17897`

有配置，直接 `ssh ziplab-5090`

# **一. 欧几里得算法（Euclidean Algorithm）**

## **定义与目的**

- **目标**：计算两个非负整数的**最大公约数（GCD, Greatest Common Divisor）**。
- **核心思想**：基于“辗转相除”原理，即用较大数除以较小数，再用余数重复此过程，直到余数为零，最后的非零余数即为 GCD

## **算法步骤**

1. **输入**：两个非负整数 $a$ 和 $b$（假设 $a>b$）
2. **迭代计算**：
    - 计算 $a÷b$ 得到余数 $r$
    - 若  $r=0$，则  $b$ 是 GCD
    - 否则，令 $a=b$， $b=r$，重复步骤
3. **终止条件**：余数为零，返回当前的 $b$
## **时间复杂度**

- **效率**：$O(log(min(a,b)))$，因每次迭代至少将数值减半。

## **扩展欧几里得算法**

- **用途**：求解贝祖(裴蜀)等式 $ax+by=GCD(a,b)$，并找到整数解 $x,y$。
	```python
	def extended_gcd(a, b):
	    """
	    扩展欧几里得算法，求解贝祖等式 ax + by = gcd(a, b) 的整数解 (x, y)
	    参数:a, b: 需要求解的整数
	    返回:gcd(a, b), x, y  使得 ax + by = gcd(a, b)
	    """
	    # 递归终止条件：如果 b == 0，gcd(a, 0) = a，且 x = 1, y = 0
	    if b == 0:
	        return a, 1, 0
	    # 递归求解 gcd(b, a % b)，以及对应的 x1, y1
	    gcd, x1, y1 = extended_gcd(b, a % b)
	    # 通过递推公式计算 x 和 y
	    x = y1
	    y = x1 - (a // b) * y1

	    return gcd, x, y
	```

- **应用**：计算模逆元（如 RSA 加密中的私钥生成）。
# **二. 幂运算（Exponentiation )**

## **定义与目的**

- **目标**：高效计算 $a^b$ ，尤其是当 $b$ 极大时（如密码学中的大数运算）

## **朴素方法**

- **直接计算**：通过  $b$ 次乘法计算 $a^b$，时间复杂度为 $O(b)$，效率极低。

## **快速幂算法（Fast Exponentiation）**

- **核心思想**：利用指数的二进制分解与平方操作，减少乘法次数。
- **步骤**：
    1. 将指数 $b$ 表示为二进制形式（如 $13=1101_2$）
    2. 通过**迭代平方**计算 $a^{2^k}$ ，并根据二进制位累乘结果
- **时间复杂度**：$O(\log b)$，显著优于朴素方法

## **快速模幂运算**

- **应用场景**：加密算法（如 RSA）中计算 $a^b \mod  m$。
- **优化**：结合快速幂与模运算性质，每一步都取模以避免数值溢出。
- **步骤**：
快速模幂运算（Fast Modular Exponentiation）是一种高效计算 $a^b \mod m$ 的方法，特别适用于大数运算（如密码学中的 RSA 算法）。它的核心思想是通过**分治**和**模运算性质**将指数分解为二进制形式，逐步计算并减少中间结果的大小。
### 1. **算法步骤（平方-乘方法）**
1. **将指数 $b$ 表示为二进制**：
   例如，$b = 13$ 的二进制是 $1101$，即 $13 = 8 + 4 + 1$。

2. **迭代计算**：
   $a^{13}=a^8 \times a^4 \times a^0 \times a^1$从最低位到最高位逐位处理：
   - 初始化结果 $\text{res} = 1$。
   - 对每一位：
     - **平方当前底数**：$a \leftarrow a^2 \mod m$。
     - 如果当前二进制位为 1，则**乘到结果**中：$\text{res} \leftarrow (\text{res} \times a) \mod m$。

3. **返回结果**：最终得到 $a^b \mod m$。
### **2. Python 代码实现**
```python
def fast_modular_exponentiation(a, b, m):
    result = 1
    a = a % m  # 确保初始 a < m

    while b > 0:
        # 如果当前位为 1，乘到结果中
        if b % 2 == 1:
            result = (result * a) % m
        # 平方底数并取模
        a = (a * a) % m
        # 右移一位（相当于 b // 2）
        b = b >> 1

    return result
```
### **3. 示例计算**
计算 $7^{13} \mod 11$：
1. $13$ 的二进制：$1101$。
2. 逐步计算：
   - $b=1$: $\text{res} = 7 \mod 11 = 7$
   - $b=0$: 平方 $a = 7^2 \mod 11 = 49 \mod 11 = 5$
   - $b=1$: $\text{res} = (7 \times 5) \mod 11 = 35 \mod 11 = 2$
   - $b=1$: 平方 $a = 5^2 \mod 11 = 25 \mod 11 = 3$,
     $\text{res} = (2 \times 3) \mod 11 = 6 \mod 11 = 6$
3. 最终结果：**6**。
### **4. 时间复杂度**
- **$O(\log b)$**：因为指数 $b$ 被分解为二进制，位数是 $\log_2 b$。
### **5. 数学推导**
- 利用了模运算的性质：
  $(a \times b) \mod m = [(a \mod m) \times (b \mod m)] \mod m$。
- 通过二进制分解，将乘幂转化为多次平方和乘法，避免直接计算大数。
### **6. 应用场景**
- 加密算法（RSA、Diffie-Hellman）。
- 大数模运算（避免溢出）。
- 竞争性编程中的高效计算需求。
# 三. RSA加密算法

## **1. 概述**
- **类型**：非对称加密
- **核心**：基于大数分解的困难性
- **用途**：加密、数字签名、密钥交换
## **2. 密钥生成**
1. 选择两个大素数 $p$ 和 $q$
2. 计算模数 $n = p \times q$。
3. 计算欧拉函数 $\phi(n) = (p-1)(q-1)$。
4. 选择公钥指数 $e$（通常为 $65537$），满足 $\text{GCD}(e, \phi(n)) = 1$。
5. 计算私钥指数 $d$，满足 $e \times d \equiv 1 \mod \phi(n)$。

- **公钥**：$(e, n)$
- **私钥**：$(d, n)$
## **3. 加密与解密**
- **加密**：
  $c = m^e \mod n$
  （$m$ 为明文，$c$ 为密文）
- **解密**：
  $m = c^d \mod n$
## **4. 数学原理**
- **欧拉定理**：
	若 $\text{GCD}(m, n) = 1$，则 $m^{\phi(n)} \equiv 1 \mod n$。
	由于 $e \times d \equiv 1 \mod \phi(n)$，有：
	$m^{e \times d} \equiv m^{k \cdot \phi(n) + 1} \equiv (m^{\phi(n)})^k \times m \equiv 1^k \times m \equiv m \mod n$
- **安全性**：依赖于大数分解的困难性

# 四. 稀疏矩阵

## 1. 理论基础

## 1. 概念和分类

+ **概念：**
	稀疏矩阵是一种特殊类型的矩阵，其中**大多数元素为零**。稀疏矩阵在实际应用中非常常见，例如图像处理、机器学习和科学计算。
+ **分类：** 根据其非零元素的分布方式：
	- **对角线稀疏矩阵：** 非零元素主要分布在对角线上。
	- **带状稀疏矩阵：** 非零元素主要分布在对角线附近的一条或多条带状区域内。
	- **块状稀疏矩阵：** 非零元素主要分布在矩阵的某些块内。
	- **非结构化稀疏矩阵：** 非零元素分布不规则。

## 2 存储格式

### a. **COO（坐标）格式（Coordinate Format）**

- **原理：**
	坐标格式将稀疏矩阵的非零元素及其在矩阵中的位置存储在三个数组中：行索引数组、列索引数组和值数组
+ **优点：**
	存储空间最少，适用于非零元素数量较少的稀疏矩阵
-  **缺点：**
	矩阵运算效率较低，因为需要遍历所有元素
- **示例**
	```
	矩阵：
	[0 0 3]
	[0 4 0]
	[0 0 0]

	COO 格式：
	行索引: [0, 1]
	列索引: [2, 1]
	值:     [3, 4]
	```
### b. **CSR格式（Compressed Sparse Row）**

- **原理：**
	CSR格式将稀疏矩阵按行存储。维护三个数组：
	1. V，用来存储矩阵中的非零元素的值；
	2. COL_INDEX，第$i$个元素记录了$V[i]$元素的列数；
	3. ROW_INDEX, 第$i$个元素记录了前$i-1$行包含的非零元素的数量，ROW_INDEX 前$n-1$个元素表示矩阵中每一行的第一个非零元素在value数组中的索引，ROW_INDEX 的最后一个元素表示矩阵一共有多少个非零元素
- **优点：**
	矩阵运算效率较高，因为可以快速定位每一行的非零元素
- **缺点：**
	存储空间比坐标格式更大
- **示例**
```
矩阵：
[0 0 3]
[0 4 0]
[0 0 0]

CSR 格式：
values:      [3, 4]
col_indices: [2, 1]
row_ptr:     [0, 1, 2]
```
### **c. CSC格式（Compressed Sparse Column）**
同CSR，但压缩列而非行
### **d. 其他格式**
- **DIA（Diagonal Format）**：适合对角线稀疏矩阵。
- **ELL（ELLPACK Format）**：适合 GPU 加速。
- **DOK（Dictionary of Keys）**：适合动态构建矩阵。

# 五. 前中后缀表达式详解

## **1. 基本概念**
表达式是数学运算的逻辑表示形式，根据运算符的位置可分为以下三种类型：

| **类型**     | **运算符位置**             | **示例**               |
|--------------|---------------------------|------------------------|
| **中缀表达式** | 运算符在操作数中间         | `A + B * (C - D)`      |
| **前缀表达式** | 运算符在操作数前面         | `+ A * B - C D`        |
| **后缀表达式** | 运算符在操作数后面         | `A B C D - * +`        |

## **2. 中缀表达式（Infix Notation）**
### **特点**
- **直观性**：符合人类阅读习惯（如 `3 + 4 * 2`）。
- **优先级依赖**：需处理运算符优先级（如乘除优先于加减）。
- **括号需求**：需用括号明确运算顺序（如 `(3 + 4) * 2`）。
### **缺点**
- **计算机处理复杂**：需额外解析优先级与括号。

## **3. 前缀表达式（波兰表达式，Polish Notation）**
### **特点**
- **无需括号**：运算符顺序直接决定运算优先级。
- **从右向左扫描**：运算时需反向处理表达式。
- **示例**：
  - `+ 3 * 4 2` 等价于中缀 `3 + (4 * 2)`。
  - `* + 3 4 2` 等价于中缀 `(3 + 4) * 2`。
### **计算方法**
1. 从右向左扫描表达式。
2. 遇到运算符时，取出后续操作数进行计算。
3. 将结果替换回原表达式，继续扫描。

## **4. 后缀表达式（逆波兰表达式，Reverse Polish Notation）**
### **特点**
- **无需括号**：运算符顺序隐含优先级。
- **从左向右扫描**：运算顺序与计算顺序一致。
- **示例**：
  - `3 4 2 * +` 等价于中缀 `3 + (4 * 2)`。
  - `3 4 + 2 *` 等价于中缀 `(3 + 4) * 2`。

### **计算方法**
1. 使用栈辅助计算。
2. 从左向右扫描表达式：
   - 遇到操作数则入栈。
   - 遇到运算符则弹出栈顶两个元素运算，结果入栈。
3. 最终栈顶为计算结果。

**代码示例（Python）**：
```python
def eval_rpn(tokens):
    stack = []
    for token in tokens:
        if token in "+-*/":
            b = stack.pop()
            a = stack.pop()
            if token == '+': stack.append(a + b)
            elif token == '-': stack.append(a - b)
            elif token == '*': stack.append(a * b)
            elif token == '/': stack.append(int(a / b))
        else:
            stack.append(int(token))
    return stack[0]

# 示例：计算 "3 4 2 * +"
print(eval_rpn(["3", "4", "2", "*", "+"]))  # 输出 11
```

## **5. 中缀表达式转后缀表达式**
### **算法步骤（使用栈）**
1. **初始化**：
   - 操作数直接输出。
   - 运算符栈用于暂存未处理的运算符。
2. **扫描规则**：
   - 左括号 `(`：入栈。
   - 右括号 `)`：弹出栈顶元素直到遇到左括号。
   - 运算符：
     - 若栈顶运算符优先级 ≥ 当前运算符，则弹出栈顶元素。
     - 当前运算符入栈。
3. **结束扫描**：将栈中剩余运算符依次弹出。

### **示例转换**
**中缀表达式**：`3 + 4 * 2 / (1 - 5)`
**步骤**：
1. 输出 `3`，栈空，`+` 入栈。
2. 输出 `4`，`*` 入栈（优先级高于 `+`）。
3. 输出 `2`，遇到 `/`，`*` 弹出（优先级相同），`/` 入栈。
4. 遇到 `(`，入栈。输出 `1`，`-` 入栈，输出 `5`。
5. 遇到 `)`，弹出 `-`，弹出 `(`。栈顶 `/` 优先级 ≥ 当前无，结束。

**后缀表达式**：`3 4 2 * 1 5 - / +`

## **6. 中缀转前缀表达式**
### **算法步骤**
1. **反转中缀表达式**：从右向左扫描。
2. **交换左右括号**：`(` → `)`，`)` → `(`。
3. **类似后缀转换**：但结果需反转最终输出。

**示例**：
中缀：`(A + B) * (C - D)`
反转后：`) D - C ( * ) B + A (`
转换过程类似后缀，最终反转得到前缀表达式：`* + A B - C D`

## **7. 三种表达式对比**
| **维度**    | **中缀表达式**    | **前缀表达式**    | **后缀表达式** |
| --------- | ------------ | ------------ | --------- |
| **运算符位置** | 中间           | 前面           | 后面        |
| **括号需求**  | 需要括号明确优先级    | 无需括号         | 无需括号      |
| **计算顺序**  | 依赖优先级和括号     | 从右向左扫描       | 从左向右扫描    |
| **计算机处理** | 复杂（需栈和优先级解析） | 较简单          | 简单（直接栈计算） |
| **应用场景**  | 人类可读的数学表达式   | Lisp 语言、符号计算 | 计算器、编译器优化 |

## **8. 实际应用**
- **计算器实现**：后缀表达式直接求值（如早期 HP 计算器）。
- **编译器设计**：将中缀表达式转换为后缀形式生成中间代码。
- **函数式编程**：前缀表达式天然契合 Lisp/Scheme 语法。

# 六. 约瑟夫环问题
- 约瑟夫环问题：n个人围成一圈，从第一个人开始报数，每次数到m的人出局，剩下的人继续从下一个位置开始重新报数，直到剩下最后一个人。要求确定**最后剩下的人的初始位置**。

## 1. 递推公式法
1. **递推公式**：
   $f(n, m) = (f(n-1, m) + m) \% n$，其中 $f(1, m) = 0$（编号从0开始）。
2. **推导思路**：
   - 当有$n$个人时，第一个出局的是位置 $(m-1) \% n$。
   - 剩下的$n-1$个人形成新环，问题转化为规模为$n-1$的子问题。
   - 子问题的解在原问题中的位置为 $(f(n-1, m) + m) \% n$。
3. **代码实现**（时间复杂度$O(n)$，空间复杂度$O(1)$）：
```python
def josephus(n, m):
    res = 0
    for i in range(2, n + 1):
        res = (n, m) + m) % i
    return res  # 从0开始编号
# 若从1开始，返回 res + 1
```
## 2. 模拟
利用列表模拟报数过程：
1. **步骤**：
   - 维护当前剩余人员的列表。
   - 每次计算待出局位置并移除对应元素，直到只剩一人。
2. **代码实现**（时间复杂度$O(n\times m)$，空间复杂度$O(n)$）：
```python
def josephus_simulate(n, m):
    people = list(range(n))
    idx = 0
    while len(people) > 1:
        idx = (idx + m - 1) % len(people)
        people.pop(idx)
    return people[0]  # 从0开始编号
```

## 3. 示例
- **输入**：$n=7$，$m=3$
- **递推过程**：
  $f(1,3)=0 → f(2,3)=1 → f(3,3)=1 → f(4,3)=0 → f(5,3)=3 → f(6,3)=0 → f(7,3)=3$
- **结果**：$3$（从$0$开始编号，对应第$4$号位置）。

## 4. 特殊情形（$m=2$）
- 当$m=2$时，可直接计算：
	- 找到最大的$2$的幂$2^k \leq n$，解为$2(n - 2^k) + 1$（从1开始编号）。

# 七、位运算技巧
- 集合可以用二进制表示，二进制**从低到高**第 $i$ 位为 $1$ 表示 $i$ 在集合中，为 $0$ 表示 $i$ 不在集合中
- 包含非负整数的集合 S 可以用如下方式「压缩」成一个数字：$f(S)=\sum_{i∈S} 2^i$
- 例如集合 $\{0,2,3\}$ 可以压缩成 $2^0+2^2+2^3=13$，也就是二进制数 $1101_{(2)}$
- 有关的库函数 (C++)
	- 集合大小 `__builtin_popcount(s)`
	- 二进制长度 `__lg(s)+1`
	- 集合最大元素 `__lg(s)`
	- 集合最小元素 `__builtin_ctz(s)`
## 1. 集合与集合
| 术语         | 集合表示                            | 位运算表示                                               | 集合示例                                            | 位运算示例                                        |
| ---------- | ------------------------------- | --------------------------------------------------- | ----------------------------------------------- | -------------------------------------------- |
| **交集**     | $A \cap B$                      | $a \& b$                                            | $\{0, 2, 3\} \cap \{0, 1, 2\} = \{0, 2\}$       | `1101 & 0111 = 0101`                         |
| **并集**     | $A \cup B$                      | $a \mid b$                                          | $\{0, 2, 3\} \cup \{0, 1, 2\} = \{0, 1, 2, 3\}$ | `1101 \| 0111 = 1111`                        |
| **对称差**    | $A \Delta B$                    | $a \oplus b$                                        | $\{0, 2, 3\} \Delta \{0, 1, 2\} = \{1, 3\}$     | `1101 ^ 0111 = 1010`                         |
| **差集**     | $A \setminus B$                 | $a \& \sim b$                                       | $\{0, 2, 3\} \setminus \{1, 2\} = \{0, 3\}$     | `1101 & 1001 = 1001`                         |
| **差集（子集）** | $A \setminus B,\ B \subseteq A$ | $a \oplus b$                                        | $\{0, 2, 3\} \setminus \{0, 2\} = \{3\}$        | `1101 ^ 0101 = 1000`                         |
| **包含于**    | $A \subseteq B$                 | $a \& b == a$或 $a \mid b == b$或$(a \& \sim b) == 0$ | $\{0, 2\} \subseteq \{0, 2, 3\}$                | `0101 & 1101 = 0101` 或 `0101 \| 1101 = 1101` |
## 2. 集合与元素

| 术语           | 集合表示                              | 位运算表示                       | 集合示例                                                     | 位运算示例                              |
| ------------ | --------------------------------- | --------------------------- | -------------------------------------------------------- | ---------------------------------- |
| **空集**       | $\varnothing$                     | `0`                         | —                                                        | —                                  |
| **单元素集合**    | $\{i\}$                           | $1 \ll i$                   | $\{2\}$                                                  | `1 << 2 = 0100`                    |
| **全集**       | $U = \{0, 1, \ldots, n-1\}$       | $(1 \ll n) - 1$             | $\{0, 1, 2, 3\}$                                         | `(1 << 4) - 1 = 1111`              |
| **补集**       | $\complement_U S = U \setminus S$ | $\sim s \& ((1 \ll n) - 1)$ | $U = \{0, 1, 2, 3\},\ \complement_U \{0, 1, 2\} = \{3\}$ | `~0110 & 1111 = 1001`              |
| **属于**       | $i \in S$                         | $(s \gg i) \& 1 = 1$        | $2 \in \{0, 2, 3\}$                                      | `(1101 >> 2) & 1 = 1`              |
| **不属于**      | $i \notin S$                      | $(s \gg i) \& 1 = 0$        | $1 \notin \{0, 2, 3\}$                                   | `(1101 >> 1) & 1 = 0`              |
| **添加元素**     | $S \cup \{i\}$                    | $s \mid (1 \ll i)$          | $\{0, 3\} \cup \{2\} = \{0, 2, 3\}$                      | `1001 \| 0100 = 1101`              |
| **删除元素**     | $S \setminus \{i\}$               | $s \& \sim (1 \ll i)$       | $\{0, 2, 3\} \setminus \{2\} = \{0, 3\}$                 | `1101 & ~0100 = 1001`              |
| **删除元素（存在）** | $S \setminus \{i\},\ i \in S$     | $s \oplus (1 \ll i)$        | $\{0, 2, 3\} \setminus \{2\} = \{0, 3\}$                 | `1101 ^ 0100 = 1001`               |
| **删除最小元素**   | —                                 | $s \& (s - 1)$              | —                                                        | 例：`1010 & 1001 = 1000`（删除最低位的 `1`） |
## 3. 遍历集合
- 设元素范围从 $0$ 到 $n−1$，枚举范围中的元素 $i$，判断 $i$ 是否在集合 $s$ 中
```c++
for (int i = 0; i < n; i++)
{
	if ((s >> i) & 1) { // i 在 s 中
	// 处理 i 的逻辑
	}
}
```
- 也可以直接遍历集合 s 中的元素：不断地计算集合最小元素、去掉最小元素，直到集合为空
```c++
for (int t = s; t; t &= t - 1) {
    int i = __builtin_ctz(t);
    // 处理 i 的逻辑
}
```
## 4. 枚举集合
1. 枚举所有集合：设元素范围从 0 到 n−1，从空集 ∅ 枚举到全集 U：
```c++
for (int s = 0; s < (1 << n); s++)
{
// 处理 s 的逻辑
}
```
2. 枚举非空子集：设集合为 s，**从大到小**枚举 s 的所有**非空**子集 sub
```c++
for (int sub = s; sub; sub = (sub - 1) & s)
{
// 处理 sub 的逻辑
}
```
3. 枚举子集（包含空集）:
```c++
int sub = s;
do
{
// 处理 sub 的逻辑
sub = (sub - 1) & s;
}
while (sub != s);
```
**原理**: 当 $sub=0$ 时（空集），再减一就得到 $−1$，对应的二进制为 $111⋯1$，再 $\&s$ 就得到了 $s$。所以当循环到 $sub=s$ 时，说明最后一次循环的 $sub=0$（空集），$s$ 的所有子集都枚举到了，退出循环
4. 枚举超集
```c++
for (int s = t; s < (1 << n); s = (s + 1) | t) {
// 处理 s 的逻辑
}
```
5. 枚举全集 U 的所有大小**恰好**为 k 的子集(**Gosper's Hack**)
```c++
void GospersHack(int k, int n)
{
    int cur = (1 << k) - 1;
    int limit = (1 << n);
    while (cur < limit)
    {
        // 处理 s 的逻辑
        int lb = cur & -cur; // 计算最低位的 1
        int r = cur + lb; // 计算进位后的数
        cur = ((r ^ cur) >> __builtin_ctz(lb) + 2) | r;
        //调整右侧的 1, 确保 1 的总数不变
        // 或：cur = (((r ^ cur) >> 2) / lb) | r;
    }
}
```
- 思路：**把最后一个 01 变成 10 ，然后把它右边的 1 全部集中到最右边即可**
- lowbit : `lb = cur & -cur;` 可以得到最小位
- 例子：GospersHack(3, 5)
```text
00111
01011
01101
01110
10011
10101
10110
11001
11010
11100
```
